# 사용 가이드

AI Builder Platform의 사용 방법을 단계별로 설명합니다.

## 🎯 기본 사용법

### 1. 플랫폼 접속

브라우저에서 [http://localhost:3000](http://localhost:3000) (개발) 또는 배포된 URL 접속

### 2. AI와 대화하며 앱 생성

왼쪽 채팅 인터페이스에서 원하는 앱을 설명하세요.

**예시 프롬프트:**
- "감정 분석 챗봇 만들어줘"
- "이미지 설명 생성기 만들어줘"
- "요약 도구 만들어줘"
- "번역기 만들어줘"

### 3. 실시간 프리뷰 확인

오른쪽에서 생성된 앱이 즉시 실행되는 것을 확인할 수 있습니다.
- 코드 수정 가능 (Sandpack 에디터)
- 실시간 반영
- 에러 확인 및 디버깅

### 4. 배포

**🚀 배포하기** 버튼을 클릭하여 별도 URL로 배포합니다.
- 고유 URL 생성 (예: `https://demo.ilhaera.com/abc123`)
- 클립보드에 자동 복사
- 누구나 접속 가능

## 💡 프롬프트 작성 팁

### 구체적으로 작성

❌ 나쁜 예: "챗봇 만들어줘"
✅ 좋은 예: "사용자가 입력한 텍스트의 감정(긍정/부정/중립)을 분석하는 챗봇을 만들어줘. Tailwind로 깔끔하게 디자인해줘."

### 기능 명시

✅ "입력창, 전송 버튼, 대화 기록이 있는 채팅 UI"
✅ "로딩 중 표시"
✅ "에러 처리 포함"

### 디자인 요청

✅ "다크 모드로"
✅ "그라디언트 배경"
✅ "모던하고 미니멀한 디자인"

## 🎨 예제 프롬프트

### 1. 감정 분석 챗봇

```
사용자가 텍스트를 입력하면 AI가 감정을 분석하는 앱을 만들어줘.

요구사항:
- 입력창과 전송 버튼
- 분석 결과를 긍정/부정/중립으로 표시
- 신뢰도 퍼센트 표시
- 다크 모드 디자인
- 로딩 상태 표시
```

### 2. 이미지 설명 생성기

```
이미지 URL을 입력하면 AI가 이미지를 설명하는 앱을 만들어줘.

기능:
- URL 입력창
- 이미지 미리보기
- AI 설명 생성
- 복사 버튼
- 반응형 디자인
```

### 3. 텍스트 요약 도구

```
긴 텍스트를 요약하는 도구를 만들어줘.

특징:
- textarea로 긴 텍스트 입력
- 요약 길이 선택 (짧게/중간/길게)
- 요약 결과 표시
- 단어 수 카운터
- 모던한 UI
```

### 4. 언어 번역기

```
한국어를 여러 언어로 번역하는 앱을 만들어줘.

기능:
- 원문 입력
- 언어 선택 (영어, 일본어, 중국어)
- 번역 결과
- 복사 버튼
- 깔끔한 디자인
```

## 🔧 고급 기능

### 코드 직접 수정

Sandpack 에디터에서 코드를 직접 수정할 수 있습니다:

1. **App.js 탭**: 메인 앱 코드
2. **proxy.js 탭**: AI API 호출 헬퍼 (자동 생성)
3. 즉시 반영되는 Hot Reload

### 에러 디버깅

콘솔 패널에서 에러 확인:
- 빨간색: 에러
- 노란색: 경고
- 파란색: 정보

### 배포된 앱 관리

배포 후:
- **🔗 열기** 버튼으로 새 탭에서 확인
- URL 공유하여 다른 사람과 테스트
- 재배포 시 새 URL 생성 (버전 관리)

## 📊 사용 제한

### 무료 티어

- ✅ 시간당 10회 AI 요청 (IP 기준)
- ✅ 무제한 프리뷰
- ✅ 무제한 배포

### Rate Limit 초과 시

```
시연 한도를 초과했습니다. 1시간 후 다시 시도해주세요.
```

→ 1시간 후 자동으로 초기화됩니다.

## 🎯 Best Practices

### 1. 반복 개선

첫 생성 후 추가 요청으로 개선:

```
1차: "챗봇 만들어줘"
2차: "더 예쁘게 만들어줘"
3차: "로딩 애니메이션 추가해줘"
```

### 2. 기능별 분리

한 번에 너무 많은 기능을 요청하지 마세요:

❌ "채팅, 번역, 요약, 이미지 분석 다 되는 앱"
✅ "채팅 기능만 있는 앱" → 테스트 → "번역 기능 추가"

### 3. 에러 처리 요청

```
"에러가 발생하면 사용자에게 친절한 메시지를 보여줘"
"로딩 중일 때는 스피너를 표시해줘"
```

## 🚀 워크플로우 예시

### 프로토타입 → 개선 → 배포

```
1단계: 기본 기능 구현
프롬프트: "간단한 챗봇 만들어줘"
→ 프리뷰에서 테스트

2단계: UI 개선

→ 다시 테스트

3단계: 기능 추가
프롬프트: "대화 기록을 저장하는 기능 추가해줘"
→ 최종 테스트

4단계: 배포
🚀 배포하기 → URL 공유
```

## 📱 모바일 지원

생성된 앱은 자동으로 반응형입니다:
- 모바일에서도 정상 작동
- 터치 인터페이스 지원
- 작은 화면 최적화

## 🔐 보안 & 프라이버시

### API 키 안전

- ✅ 사용자는 API 키를 몰라도 됨
- ✅ 모든 요청은 Proxy 서버를 통해 안전하게 처리
- ✅ 클라이언트에 API 키 노출 없음

### 데이터 저장

- ❌ 대화 내용은 저장되지 않음
- ❌ 생성된 코드는 서버에 저장되지 않음 (배포 시만 임시 저장)
- ✅ 완전한 프라이버시

## 🆘 자주 묻는 질문 (FAQ)

### Q: 생성된 앱이 작동하지 않아요

A: 콘솔 패널에서 에러를 확인하고, AI에게 에러 메시지를 보여주며 수정 요청하세요.

### Q: 더 복잡한 앱을 만들 수 있나요?

A: 네! 여러 번 대화하며 점진적으로 기능을 추가할 수 있습니다.

### Q: 배포된 앱의 URL을 변경할 수 있나요?

A: 현재는 고유 ID가 자동 생성됩니다. 커스텀 도메인은 추후 지원 예정입니다.

### Q: 생성된 코드를 다운로드할 수 있나요?

A: Sandpack 에디터에서 코드를 복사하여 로컬에 저장할 수 있습니다.
